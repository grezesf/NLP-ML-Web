Bad partitioning of the Hard Disk might be a possible reason...

When you partition your hard disk you should first be understanding the limitations of hard disk structures and file systems.

Are you using MBR partitioning? Yes... Then you must know that the sum of primary and extended partitions for a disk can't exceed 4. Extended partitions consist of logical partitons partitions i.e. one or more logical partitons partitions constitute an extended partition. If you use dual boot take care and install Windows OS in a primary partition, while you may install ubuntu Ubuntu in primary or in logical partition(s), a primary partition is preferred for a /boot partition though.

So... May be you'll have to count your partitions... and then partition your HDD wisely abiding by the rules if this is the reason why you've been having the problem. For this GParted or other hard disk partitioning utilities may prove useful to you. So try to understand the partitioning (if you haven't already) and yes... good luck!

Elaboration to my experience upon which I based answer:

A few years back, this is how I planned my partitions:

  1. a 100M sda1 (for windows' system partition, ntfs),
  2. an ntfs formated formatted sda2 for my windows 7 files,
  3. sda3, ntfs and
  4. sda4, extended partition with just two logical partitions; one for the root (ext4) and the other linux-swap.

So those were my 4 partitions. Then I installed Windows 7 to sda2 (i knew it would overwrite the bootloader) and expected sda1 would be used by this installation (yes I was wrong thinking so) but rather it created another 100M partition. Note that this install further split my sda2 partition into two separate partitions. (bad partitioning from myself) Thus I was left with 5 partitions (primary + extended) in total. Then I reinstalled Ubuntu (I had to, I didn't have had success with command line rescue for the grub or the grub2 reinstallation re-installation) successfully. But still my system was unbootable. So I had to repartition my HDD wisely again.

Yes for MBR partitioning style the partition managers won't allow you to have more than 4 primary + extended partitions; I never opposed that @nyarlathotep (in the comments). So that's what I lived... but did learn somthing something wrong? (comments are and will be appreciated, for we're here to share and learn)... Thank you.

Due to flaky internet connection?

I read the description of the problem in the comments by @Shawn after I posted my answer (my mistake having not read earlier and focusing on partitioning problems). The problem was actually during installation with internet connections up which according to him was flaky; and I don't know whether the installation was successful or not. And in the end he mentions he did a reinstall without connecting to the internet which worked well.

Yes, it's better to detach the internet cable (wireless connections will have its own issues, so better avoid that too) during Ubuntu installation. You can always do the updates and upgrades after the system installation over which is far more simple and reliable.

@Tesla had a different reason; problem with display-manager/login-manager; and a different solution for that.
