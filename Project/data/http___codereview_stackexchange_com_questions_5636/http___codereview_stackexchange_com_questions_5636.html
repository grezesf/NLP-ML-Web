Am I using too many scripts? [closed]

up vote 2 down vote favorite

My site is pinglio.com. If you look at my source code you'll notice that I'm using 4 stylesheets and over 10 javascript files. My site has been a little slow lately, and, apart from an increase in traffic, I think I may have too much loaded on it.

Am I using too many scripts? Any general suggestions on my site setup? Feedback would be greatly appreciated.

It's important to get your caching settings correct(ETags, Expires, etc.). The YSlow docs will point you in the right direction. This makes a significant difference to page performance, in my experience.

If you're loading your 'fancy-box' script on every page, it should only be pulled down once (when a visitor first visits your site), and then pulled from cache for every subsequent page request.

Yes, you should absolutely combine your external files where possible, as there is an overhead to requesting each file individually.

If a Javascript file is on every page (or most pages) then combine them all together. You should also minify them using Google's Closure Compiler, which will bring down the file size even more.

Similarly for CSS, use Yahoo's YUI Compressor (a Java app) to minify CSS. There is an online version here.

Sometimes you may need a bunch of Javascript or CSS specific to one or two pages, in which case you should keep that file separate and include it on the relavant pages. I can't think of a situation where you'd ever need more than 2 JS and 2 CSS files on a page (not including external scripts like Google Analytics).

From a quick look it seems most of your background images are being tiled, which means they can't be combined to CSS sprites. Images in the body of the page should stay as images (not backgrounds) as they are part of the content.

Aside from that, check that these files are being cached by the browser. From a quick look it appears they are (you have a Last-Modified header). Ignore YSlow's ETag rule, it's wrong for so many reasons.
