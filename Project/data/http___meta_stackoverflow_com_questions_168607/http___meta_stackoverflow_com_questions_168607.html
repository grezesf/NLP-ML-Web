The API which we pull these from was undergoing maintenance yesterday that lasted far longer than expected. Out of that are 2 notes:

  1. We very rarely need to do a maintenance like this, only a major interruption in service causes a need.
  2. We made substantial speed improvements on the process so future outages will be far less in duration.
