It's difficult to understand your question. Do you mean that you want to copy the data in a table into new rows, but update DATE values?

For example, is the following what you're trying to do?

ORIGINAL TABLE_1:
TABLE_ID | VALUE_1    | VALUE_2         | DATEFIELD
1        | 'Somedata' | 'Someotherdata' | 2012-05-01 ...

And you want to produce:

UPDATED TABLE_1:
TABLE_ID | VALUE_1    | VALUE_2         | DATEFIELD
1        | 'Somedata' | 'Someotherdata' | 2012-01-01 08:59...
2        | 'Somedata' | 'Someotherdata' | 2013-05-08 09:00...

For instance?

In that case, you can just run a regular insert, using the source data as the source for the insert. Copying all other values directly save for DATEFIELD where you just manually use whatever value you want.

To create the scenario above in the code segments, you can for instance use this script (you have to come up with the filters for which range of data you want, since we don't know):

INSERT INTO TABLE_1 (VALUE_1, VALUE_2, DATEFIELD)
SELECT VALUE_1, VALUE_2, GETDATE() 
FROM TABLE_1

But as Raj said, you need to open up your question more. At this point it's not even logically clear what you're even asking for, let alone the technical specs for what you want.
