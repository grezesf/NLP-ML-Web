Module personalized settings

up vote 0 down vote favorite

I've written a module. I've also written a settings page for the admin using hook_admin(). I also want to have user specific settings for this module.

Is there a drupal hook to implement this, or do I have to build a standard form which detects the user and on submit write some data into a table?
Of course there must be a link in the menu to call this settings form, but should I also use something like hook_admin()?

There is no hook hook_admin in Drupal.

You can implements hook_menu to create menu item and if you want to create a form for your module configuration, then you need to implement hook_form in your module.

You need to create a custom module and implements hook hook_menu in your module to generate admin menu for your module.

In mymodule.module file

/**
 *Implementation of hook_menu
 */
function mymodule_menu() {

  $items = array();

  $items['admin/config/system/mymodule'] = array(
    'title' => 'mymodule',
    'description' => 'Configure mymodule Curation Module',
    'page callback' => 'drupal_get_form',
    'page arguments' => array('mymodule_settings_form'),
    'access arguments' => array('administer site configuration'),
    'file' => 'mymodule.admin.settings.inc',

  );
  return $items;
}

You need to create mymodule.admin.settings.inc file in your module directory and add mymodule_settings_form in file.

function mymodule_settings_form() {

  $form['msg'] = array(
    '#type' => 'textfield',
    '#title' => 'Name', 
  );

  return $form;
}

If you want to save data into your custom database table, then you need to create your form submit callback and otherwise you can use return system_settigs_form($form) instead of return $form in your mymodule_settings_form() function.

If you need to keep settings per user, just add his UID to a form structure.

funcrion mymodule_settings($form, &$form_state) { // Or whatever your function is named now
  global $user;
  $form['mymodule_user_dependent_value'][$user->uid] => array(
    '#type' => 'textfield',
    '#title' => 'Name',
  );

If you have many user-dependent ones, group them together:

funcrion mymodule_settings($form, &$form_state) { // Or whatever your function is named now
  global $user;
  $form['mymodule_users_settings'][$user->uid] => array(
    'first_name' => array(
      '#type' => 'textfield',
      '#title' => 'First Name',
    ),
    'last_name' => array(
      '#type' => 'textfield',
      '#title' => 'Last Name',
    ),
  );

That way, your setting variable will be an array keyed with user IDs.

You could use hook_menu for that.

Provide a custom page that takes values from the user.

This hook enables modules to register paths in order to define how URL requests are handled. Paths may be registered for URL handling only, or they can register a link to be placed in a menu (usually the Navigation menu). A path and its associated information is commonly called a "menu router item". This hook is rarely called (for example, when modules are enabled), and its results are cached in the database.

/**
 *Implementation of hook_menu
 */
function mymodule_menu() {

    $items = array();

    $items['example'] = array(
      'title' => 'Example Page',
      'page callback' => 'example_page',
      'access arguments' => array('access content'),
      'type' => MENU_SUGGESTED_ITEM,
    );

    return $items;
}
