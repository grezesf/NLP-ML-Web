Well, you're not alone at least.

MSI GE40 here, with GTX 760M. Exact same issues, optirun works fine and makes a difference, but I can't turn off the discrete card (and it's using most of the power). No /proc/acpi/bbswitch - bbswitch doesn't like to load:

[119624.055053] bbswitch: version 0.7
[119624.055059] bbswitch: Found integrated VGA device 0000:00:02.0: \_SB_.PCI0.GFX0
[119624.055062] bbswitch: cannot find ACPI handle for VGA device 0000:01:00.0
[119624.055064] bbswitch: No discrete VGA device found

If I find a solution I'll be sure to come back here...

Edit: I found this thread,

https://github.com/Bumblebee-Project/bbswitch/issues/65

which tells us it's a kernel bug from kernel 3.8.5-1. Hopefully there'll be a working kernel release again soon. Read that thread if you want to compile up your own kernel, or back the kernel to 3.8.4.

I had the same problem. Was using Ubuntu 12.04 LTS with 3.8.0-31-generic kernel (Ubuntu 13.04's kernel). I was able to get optirun and primusrun to both work, but bbswitch will not switch off the discrete GPU automatically after usage.

I managed to get bbswitch to work in the end, by installing the latest v3.11 kernel. Followed the steps on this page: http://itsfoss.com/linux-kernel-3-11-install-in-ubuntu/
