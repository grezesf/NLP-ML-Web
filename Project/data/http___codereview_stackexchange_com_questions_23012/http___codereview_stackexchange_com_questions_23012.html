The CSS that was created (i.e. the position, min-height, and padding) is not anything you have specified in your SCSS. So it must be boilerplate generated by the mixins you're using, e.g. innerRow() and column(). I expect if you look at the Foundation source SCSS for those mixins you will see they'll be using @extend internally and hence generating the large selectors. Either that or there's a bug in Foundation.
