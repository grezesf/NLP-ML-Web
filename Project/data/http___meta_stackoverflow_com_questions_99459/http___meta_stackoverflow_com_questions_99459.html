Pressing Enter should simply insert a space when adding wildcard tags

up vote 2 down vote favorite

When I enter in something into the favorites or ignored input i get something that looks like this:

When I press Enter after this point, the system pulls up the dropdown, chooses the first highlighted and then adds a space for me to add another tag. Note that the suggestions ignore the trailing dash..

So if I then type a * for a wildcard after I typed jquery- (so I now have jquery-*), this is also ignored by the autocompletion, leaving the previous suggestions intact. If I then press Enter it will again insert the selected suggestion from the dropdown and append a space.

I suggest that when a user types a * the autocomplete should not match anything, and when the user presses Enter it should just add a space next to the text that was typed (jquery-* my example) and allow for another tag to be entered.
