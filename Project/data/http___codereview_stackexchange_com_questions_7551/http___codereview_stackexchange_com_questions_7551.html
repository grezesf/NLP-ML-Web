A couple notes on design:

  1. Split the code up into different objects that handle specific tasks. You don't need to split your code up into different files, because it's not that long (yet), but you can split the functionality into objects, like this:

    FoldersManager = {

  init: function() {
    $("#add_folder").click(function () {
      $('#new_folder_modal').modal('show');       
    });

    ...
  }

};

  2. Put code that manipulates the DOM into a $.ready() block, which guarantees that the DOM will be ready when your code executes:

    $.ready(function() {
  FoldersManager.init();
  FilesManager.init();
  Paginator.init();
});

  3. As ericW mentioned, don't use global variables. They can conflict with global variables from another file which could result in hard-to-find bugs. You can stick them inside the new objects using the following pattern:

    $.ready(function() {
  FoldersManager.init();
  FilesManager.init();
  Paginator.init();
});

FoldersManager = (function() {

  var folders_tree;
  var data_per_folder = [];
  var current_folder = {};

  return {

    init: function() {
      $("#add_folder").click(FoldersManager.openAddFolderDialog);

      ...
    },

    openAddFolderDialog: function() {
      $('#new_folder_modal').modal('show');
    }

  };

});

And some minor things:

  1. Why are you doing things like this:

    $(".nav li a").attr({
    "onclick": "topbarItem_OnClick(this.href)"
});

    You can use the same $.click() function you were using for the anonymous functions:

    $(".nav li a").click(function() {
  // or change the function to use the href attribute directly
  // instead of taking it as a param
  topbarItem_OnClick($(this).attr('href'));
});

  2. This:

    if (data_per_folder[current_folder.attr('id')] == undefined)

    can be simplified to:

    if (!data_per_folder[current_folder.attr('id')])

    See http://james.padolsey.com/javascript/truthy-falsey/ for more info.

People will probably throw things at me for this answer, but here goes.

This advice goes mostly for languages like JS, but has some portability.

In a function, you are typically dealing with an object of primary concern, sometimes it is 'this' -- but other times it can be a concern global to the function, or a concern local to a function part....

To make the code more readable, name such objects generically, like o, or it. Make a comment to what that symbol really means/is at its declaration if you have to. Even if you are dealing with a global var like G.some_long_name, create a local shortname for it:

var o=G.some_long_name;

There are trade-offs to this method though - finding refs is not as easy with your editor. But compressed code like this is much easier to read, and makes copy/paste code duplication easier.

Next, do something to your parameter names to make them distinguishable from locals. I put a tail on them, and make them generic too when I can, using o_ for object, and n_ for name, i_ for index.

Use unix_style for private functions, and camelCase for public, unless you are extending a core type prototype, then you might want to use String.prototype.Has, instead of String.prototype.has -- depending on if you want to accentuate a source where something might have gone wrong versus having your extension blend in.

Name your class member vars like _member.

Repetitive code like this should be declared as an object, unless it is going to be changing dynamically:

data_per_folder[current_folder.attr('id')]

Use

var f_data=data_per_folder[current_folder.attr('id')];
f_data.something();

And here is a cool trick, instead of 

$(".pagination li.active").each(...

How about

var items=".pagination li.active".$();
items.each(...
items.ect(..

using 

String.prototype.$=function (){return $(''+this);}

You can even make a builder to dynamically create/manage the selectors you are using, a type of meta-selector:

String.prototype.$$=function (){
  var me=''+this;
  var s='';
  if (me=='coolbeans'){
     s+=condition?' #something':'#somethingelse';
     // more...
  }
  // magic
  return s.$();// as per above
}
// later
var dynamicthings='coolbeans'.$$();
